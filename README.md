# Effective Mobile — Docker / Compose Test

Описание решения тестового задания: nginx reverse proxy + внутренний HTTP backend в docker-сети.

---

## Идея решения

Целью решения было **не просто запустить контейнеры**,  
а **описать и зафиксировать границы системы**, внутри которых сервисы
могут стабильно и воспроизводимо работать.

Вся система описана декларативно и передана под управление Docker daemon.

---

## Архитектурный подход

Система состоит из двух логически разделённых компонентов:

- **nginx** — граничный сервис 
- **backend** — внутренний сервис 

Принципы:
- внешний доступ разрешён **только через nginx**
- backend **не публикует порт наружу**
- взаимодействие сервисов происходит **внутри docker-сети**
- система описана как состояние, а не как набор ручных команд

---

## Схема взаимодействия

```text
Client
  |
  v
nginx (port 80) 
  |
  v
backend (port 8080, internal)
```

---

## Границы системы

### Сетевая граница
- nginx публикует порт `80`
- backend доступен только внутри docker-сети
- backend не имеет проброса портов на хост

### Граница ответственности
- nginx отвечает только за приём и проксирование HTTP-трафика
- backend отвечает только за обработку запросов

### Граница среды
- среда backend зафиксирована через `Dockerfile`
- система сервисов описана через `docker-compose.yml`

### Наблюдения 
compose используется как контакт системы 
Dockerfile фиксирует среду выполнения
docker-compose.yml фиксирует границы и взаимодействие сервисов
Docker daemon отвечает за жизненный цикл системы
Контейнеры не управляются вручную 
система описана один раз и воспроизводится целиком.
